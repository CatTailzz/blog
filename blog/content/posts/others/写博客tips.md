# 不该写什么？

- 关于平时的一些有大量解决方案的小问题==不需要记录==
- 对于一些框架和工具的使用指南==不需要记录==

# 要写一些什么？

- 记录解决问题的流程
- 记录网上资料不多的内容
- 记录最佳实践、调优、解决方案等

# 写博客的好处多多

- 输出，能输出是掌握知识点的必要条件
- 展示，给其他或者给自己看，形成正反馈
- 锻炼，可以锻炼表达能力和深挖能力，并形成更广阔的视野
- 记录回顾

# 该怎么写？

- 要写明【起因】-【问题根源】-【解决思路】
- 多写一些自己实践部分，直接贴核心部分代码并带一些注释
- 不要假定读者能理解背景，尽量把全链路和需求解释清楚
